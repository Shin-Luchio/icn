# ICN 대회 운영 MVP 실행안 (1~2주)

## 0) 권장 기술 스택 (요청사항 반영)
- 프론트/프로모션: **Next.js**
- DB/인증/스토리지: **Supabase**
- 결제: **무통장입금 + 운영자 수동 확인**
  - 입금자명 규칙 예시: `홍길동0101` (참가자명 + 휴대폰 뒤 4자리)
- 알림: **SOLAPI 또는 CoolSMS** (알림톡 우선, 실패 시 SMS fallback)
- 이메일: **AWS SES**

---

## 1) 기능 재정의: 반드시 / 선택 (MVP 스코프)

### 반드시 (1~2주 내 현실 스코프)
1. **대회 홍보 페이지**
   - 대회 개요, 일정, 장소, 규정, 종목, 시상, 문의
   - 공지 목록
   - 후원사 섹션(텍스트/로고)

2. **참가 신청 + 결제(무통장입금)**
   - 참가 신청 폼 제출
   - 신청 직후 입금 안내 노출/발송
   - 운영자 수동 입금확인 후 결제완료 처리

3. **참가자 관리(백오피스)**
   - 신청 목록/검색/필터
   - 상태 관리: 신청, 입금대기, 결제완료, 환불
   - CSV 다운로드(엑셀 열기 가능)

4. **안내 발송**
   - 트리거 메시지: 신청완료, 입금안내, 결제확정, 대회 D-1, D-day
   - 채널: 알림톡/SMS + 이메일

5. **운영 안정성 최소 요건**
   - 개인정보 최소 수집
   - 보관기간 정책 정의 및 고지
   - 관리자 접근 통제(로그인 + 권한)

### 선택 (MVP 이후)
1. 갤러리(고급 레이아웃/영상)
2. 쿠폰/할인/조기등록 요금제
3. 자동 입금 매칭(가상계좌/PG 연동)
4. 고도화 통계 대시보드(퍼널/전환율)
5. 예약 발송 고급 기능(A/B 테스트, 세그먼트 자동화)

---

## 2) 백오피스 설계 원칙: “커스텀 최소화”

> 목표: 빠른 운영성과(검색/필터/엑셀/발송)

### 꼭 필요한 화면 5개
1. **대회 관리 (CRUD)**
   - 상태: 공개/비공개
   - 기본정보: 일정, 장소, 종목, 규정 링크, 참가비, 입금계좌

2. **신청자 목록**
   - 검색: 이름, 휴대폰, 신청번호
   - 필터: 대회, 종목, 신청일, 결제상태
   - 상세 보기 + 메모

3. **결제 상태 관리**
   - 상태 전이: 입금대기 → 결제완료 / 환불
   - 처리자/처리시각 로그 저장

4. **발송 센터**
   - 템플릿 선택
   - 대상 필터 (예: 결제완료자만)
   - 즉시 발송 (MVP), 예약 발송은 선택

5. **데이터 내보내기**
   - CSV 다운로드
   - 필터 결과 그대로 내려받기

### UI/개발 우선순위 권고
- 백오피스는 디자인보다 **처리속도와 정확성** 우선
- 표/필터/엑셀/일괄 상태변경이 핵심
- 커스텀 컴포넌트 최소화 (기본 테이블 + 폼 중심)

## 3) 운영 내구성 체크리스트

### 개인정보/보안
- [ ] 필수항목만 수집 (이름, 연락처, 종목, 결제확인 정보)
- [ ] 민감정보(주민번호 등) 수집 금지
- [ ] 개인정보처리방침/이용약관/수집동의 고지
- [ ] 보관기간 정책 정의 (예: 대회 종료 후 6~12개월)
- [ ] 관리자 계정 2FA 또는 강한 비밀번호 정책
- [ ] Supabase RLS로 관리자/일반 사용자 접근 제어

### 결제/정산
- [ ] 입금자명 규칙을 신청 완료 화면/문자/이메일에 반복 안내
- [ ] 입금확인 SLA 정의 (예: 영업일 기준 24시간)
- [ ] 환불 정책/기준일 공지
- [ ] 수동 확인 로그(누가/언제/왜) 기록

### 발송/커뮤니케이션
- [ ] 알림톡 실패 시 SMS fallback
- [ ] 템플릿 변수 누락 검사(이름/일정/장소)
- [ ] 대량 발송 전 샘플 발송(운영자 본인)
- [ ] 발송 이력 저장(성공/실패 코드)

### 장애/운영
- [ ] 오류 모니터링(서버 로그, 발송 실패율)
- [ ] 관리자 액션 로그(상태 변경/삭제)
- [ ] 정기 백업(Supabase backup/export)
- [ ] 공지사항으로 긴급 안내 가능한 운영 프로세스

## 4) 2주 실행 로드맵 (권장)

### 1주차
- Day 1~2: DB 스키마/인증/RLS, 기본 라우팅
- Day 3~4: 홍보 페이지 + 신청 폼 + 상태 저장
- Day 5: 백오피스 목록/검색/필터/상태변경

### 2주차
- Day 6~7: 발송 연동(SOLAPI/CoolSMS, SES), 템플릿 구성
- Day 8: CSV 다운로드 + 운영 로그
- Day 9: 정책 페이지(약관/개인정보), 보안 점검
- Day 10: 리허설(신청→입금확인→확정→D-1 발송)

---

## 5) 최소 데이터 모델(초안)
- `competitions`: 대회 기본정보
- `events/divisions`: 종목/부문
- `participants`: 참가자 기본정보
- `applications`: 신청서 + 상태
- `payments`: 입금확인 상태/처리로그
- `message_templates`: 알림 템플릿
- `message_logs`: 발송 결과 로그
- `admin_action_logs`: 관리자 작업 감사 로그

해당 구조는 Supabase(Postgres)에서 시작하기 적합하며, MVP 범위에서 확장성/운영성을 함께 확보할 수 있습니다.
